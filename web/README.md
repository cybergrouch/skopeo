# Skopeo Web

The Skopeo web client — sign-up, sign-in, and the post-signup **pending
assessment** screen. React + TypeScript (Vite), Tailwind + shadcn/ui, Firebase
Auth, and a typed TanStack Query client generated from the backend's OpenAPI
spec with [orval](https://orval.dev).

## Stack

| Concern        | Choice                                                          |
| -------------- | -------------------------------------------------------------- |
| Build / dev    | Vite                                                           |
| UI             | Tailwind CSS v4 + shadcn/ui (components in `src/components/ui`) |
| Routing        | React Router                                                   |
| Data fetching  | TanStack Query, hooks generated by orval                       |
| Auth           | Firebase Auth (email/password + Google)                        |
| Hosting target | Firebase Hosting                                               |

## Prerequisites

- Node 22+ and npm
- A Firebase project with **Authentication** enabled (Email/Password + Google)
- The Skopeo backend running (defaults to `http://localhost:8080`)

## Setup

```bash
cd web
npm install
cp .env.example .env.local   # fill in your Firebase web app config
npm run dev                  # http://localhost:5173
```

`.env.local` holds the **public** Firebase client config (safe in the browser);
the Firebase secret stays server-side in Firebase. In dev, `/api` is proxied to
`http://localhost:8080`, so the browser talks to a single origin and the Firebase
ID token rides along to the backend, which verifies it against Firebase's JWKS.

## Scripts

| Command                | Description                                                |
| ---------------------- | ---------------------------------------------------------- |
| `npm run dev`          | Dev server (regenerates the API client first)             |
| `npm run build`        | Type-check + production build (regenerates the API client) |
| `npm run typecheck`    | `tsc` only (regenerates the API client)                   |
| `npm run lint`         | ESLint                                                     |
| `npm run api:generate` | Regenerate the typed API client from the OpenAPI spec     |

## Generated API client

`npm run api:generate` (also run automatically before `dev`, `build`, and
`typecheck`) reads `../src/main/resources/openapi/documentation.yaml` — the
backend's hand-maintained, test-verified spec — and emits typed TanStack Query
hooks into `src/api/generated/` (gitignored). All requests go through the axios
mutator in `src/api/axios.ts`, which attaches the current Firebase ID token.

Config lives in `orval.config.ts`. Because the spec is the single source of
truth, a backend contract change surfaces here as a type error.

## Sign-up flow

1. The user authenticates with Firebase (email/password or Google) → gets an ID token.
2. The client calls `POST /api/v1/users` (idempotent) to provision the Skopeo
   profile. Identity/email come from the verified token, not the request body.
3. The user lands on **`/pending`** — until an administrator assigns an initial
   rating, the account is *pending assessment* and ineligible for match fixtures.

## Routes

- `/signup` — create account
- `/login` — sign in
- `/pending` — authenticated landing; shows pending-assessment state or the
  assigned rating(s)
